================================================================================
STUDENT LEARNING PROFILE (schema overview)
================================================================================
StudentLearningProfile contains:
- _meta, student (id, name, targetScore, daysToExam, …)
- archetype (behavioral inference ONLY — never self-report)
- analytics (6 dimensions + overallHealthScore)
- wellbeing (burnoutRisk, recommendedTone)
- chapterMastery: Record<Subject, Record<chapterId, ChapterMasteryEntry>>
- sessionHistory: SessionRecord[]
- nextAction: NextActionOutput
- recoveryPlans: RecoveryPlan[]
- targetConfig: TargetConfiguration
- adaptiveMsg: AdaptiveMessaging
- blueprint: Record<Subject, Record<chapterId, { marks, name }>>

Subjects: "math" | "science" | "social"

SessionRecord must capture:
- duration, accuracy, hintsUsed, retriesOnWrong, completedPlan
- panicSignal (score drops >15% vs practice avg under time pressure)
- engineType (adaptive | recovery | timed_test | pyq_practice | …)

================================================================================
ENGINE PIPELINE (strict execution order)
================================================================================
profile (StudentLearningProfile)
  │
  ├─► [1] scoreProjectionEngine(mastery, blueprint)
  │         Output: ScoreProjectionOutput
  │         Formula: Σ mastery[ch] × blueprintMarks[ch] / 100 per subject
  │
  ├─► [2] studentArchetypeEngine(sessions, projection)
  │         Output: ArchetypeOutput
  │         CRITICAL: inferenceMethod = "behavioral" only
  │         Archetypes: struggling | average | topper
  │
  ├─► [3] recoveryEngine(mastery, blueprint, sessions)
  │         Output: RecoveryEngineOutput (marks at risk, action plans)
  │
  ├─► [4] targetGapEngine(targetScore, projection, mastery, blueprint)
  │         Output: TargetGapOutput
  │         ROI = marksGain / estimatedHours
  │
  ├─► [5] momentumEngine(sessions)
  │         Output: MomentumOutput (streak, trend, badge)
  │
  └─► [6] nextActionEngine(recovery, target, momentum, archetype, sessions)
            Output: NextActionOutput ← Aura signature experience

Also: computeAnalytics(sessions, archetype) → AnalyticsState

Orchestrator: runAllEngines(profile) → AuraEngineOutputs

================================================================================
NEXT ACTION DECISION LOGIC
================================================================================
1. Critical recovery (mastery < 50 AND blueprintMarks >= 6) → recovery session
2. Else streak at risk / skip day → easiest high-ROI chapter
3. Else target gap > 10% → highest ROI chapter from TargetGapEngine
4. Else topper mode → precision drill on almost-mastered chapter

NextActionOutput must include:
- recommendedAction, subject, chapter, sessionType
- estimatedGain (e.g. "+4.2 marks"), timeRequired, urgency, confidence
- rationale + optional followUp chain

================================================================================
ANALYTICS DIMENSIONS (Phase A4)
================================================================================
Score 6 dimensions from session history:
- consistency, accuracy, recovery, momentum, discipline, confidenceStability
confidenceStability distinguishes panic-driven vs stable performers at same %.

================================================================================
ADAPTIVE MESSAGING
================================================================================

onPanicDetected, onTopper, onStruggling

Tone follows wellbeing.recommendedTone:
recovery | optimization | challenge | reassurance
