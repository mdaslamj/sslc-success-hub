══════════════════════════════════════════
WHAT AURA IS
══════════════════════════════════════════

Aura is an adaptive learning system for Class 10 students preparing for Karnataka SSLC board exams (Math, Science, Social Science — 80 marks each).

It is NOT a simple dashboard. It is an AI academic operating system.

The core philosophy:
- Every number must be actionable, not decorative
- Aura knows the student's situation better than the student does
- It never asks "which type of student are you?" — it infers silently
- The dashboard must make sense within 3 seconds of landing

6 ENGINES — BUILD IN THIS EXACT ORDER
══════════════════════════════════════════

E1 — ScoreProjectionEngine   src/engines/scoreProjection.ts
  Formula: Σ mastery[ch] × blueprintMarks[ch] / 100
  Inputs:  chapterMastery, blueprint
  Outputs: bySubject{predicted,max,pct}, total, totalMax, pct, grade
  STATUS:  → BUILD THIS FIRST. Everything depends on it.

E2 — StudentArchetypeEngine  src/engines/studentArchetype.ts
  CRITICAL: NO self-report. Infer from 8 behavioral signals only:
  sessionCompletionRate, accuracyTrend, panicIndex, streakDiscipline,
  helpSeekingFrequency, retryBehavior, examVsPracticeGap, recoverySpeed
  Outputs: archetype (struggling|average|topper), dashboardTone, layoutDensity

E3 — RecoveryEngine          src/engines/recovery.ts
  Key output: "You are losing 8 marks due to Electricity" — not just "weak chapter"
  Outputs: marksAtRisk, recoverableMarks, sessionsNeeded, urgency, recoveryProbability, actionPlan[]

E4 — TargetGapEngine         src/engines/targetGap.ts
  ROI = gainPossible / hoursEstimate
  Outputs: gap, rankedChapters by ROI, fastestPath, estimatedHours, reachableBy

E5 — MomentumEngine          src/engines/momentum.ts
  Outputs: streak, trend (up/flat/down), score (0–100), badge, weeklyPattern

E6 — NextActionEngine        src/engines/nextAction.ts   ← AURA'S SIGNATURE
  Unifies ALL engines into ONE recommended action.
  Decision tree:
    1. Critical chapter (mastery <50%, blueprintMarks ≥6)? → Recovery session
    2. Streak at risk (missed yesterday)?                  → Easiest high-ROI chapter
    3. Target gap >10%?                                    → Highest ROI chapter
    4. Topper mode                                         → Precision drill on near-mastered chapter
  Outputs: recommendedAction, estimatedGain, timeRequired, urgency, confidence (0–1), followUp

  runAllEngines(profile) → { projection, archetype, recovery, target, momentum, nextAction }
  Location: src/engines/pipeline.ts

══════════════════════════════════════════
THREE STUDENT ARCHETYPES
══════════════════════════════════════════

STRUGGLING (below 60%):
  Tone:    "You can still pass strongly."
  Layout:  Simplified. Focus on recovery. Emotional reassurance.
  Shows:   recoveryPath, easiestWins, dailyStreak

AVERAGE (60–85%):
  Tone:    "You are capable of 85+."
  Layout:  Standard. Weak-area targeting. Consistency tracking.
  Shows:   subjectBalance, weakAreas, targetGap

TOPPER (85–100%):
  Tone:    "You are within reach of state rank."
  Layout:  Advanced. Precision gaps. Rank prediction.
  Shows:   precisionGap, rankProjection, efficiencyScore
