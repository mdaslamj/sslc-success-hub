AURA — Karnataka SSLC Intelligence Layer v2
BUILD PROMPT (Cursor / Lovable)

================================================================================
PURPOSE
================================================================================
Implement the Aura intelligence layer for Karnataka SSLC Class 10 board prep.
Aura is a deterministic, profile-driven engine stack — not chat UI magic.

Cursor builds the engines and profile pipeline.
Lovable consumes engine outputs in dashboard / planner / practice surfaces.

================================================================================
SOURCE OF TRUTH (never bypass)
================================================================================
1. StudentLearningProfile JSON
   Path: src/data/StudentLearningProfile.json

2. TypeScript contracts (all 6 engines + shared types)
   Path: src/types/aura-engine-contracts.ts

RULE: Engines NEVER communicate through React UI state.
      Read/write ONLY via StudentLearningProfile + pure engine functions.

================================================================================
ENGINE EXECUTION ORDER (strict — do not reorder)
================================================================================
profile (StudentLearningProfile)
  │
  ├─► [1] scoreProjectionEngine(mastery, blueprint)
  │         └─► ScoreProjectionOutput
  │         Formula: Σ mastery[ch] × blueprintMarks[ch] / 100 per subject
  │
  ├─► [2] studentArchetypeEngine(sessions, projection)
  │         └─► ArchetypeOutput
  │         CRITICAL: behavioral inference ONLY — no self-report
  │
  ├─► [3] recoveryEngine(mastery, blueprint, sessions)
  │         └─► RecoveryEngineOutput
  │
  ├─► [4] targetGapEngine(targetScore, projection, mastery, blueprint)
  │         └─► TargetGapOutput
  │         ROI = marksGain / estimatedHours
  │
  ├─► [5] momentumEngine(sessions)
  │         └─► MomentumOutput
  │
  └─► [6] nextActionEngine(recovery, target, momentum, archetype, sessions)
            └─► NextActionOutput  ← Aura signature: "what to do next"

Also run: computeAnalytics(sessions, archetype) → AnalyticsState

Orchestrator: runAllEngines(profile) → AuraEngineOutputs

================================================================================
NEXT ACTION DECISION LOGIC
================================================================================
1. Critical recovery chapter (mastery < 50, blueprintMarks >= 6) → recovery session
2. Else streak at risk / skip day → easiest high-ROI chapter
3. Else target gap > 10% → highest ROI chapter from TargetGapEngine
4. Else topper mode → precision drill on almost-mastered chapter
